   - Create a `spriteCommitPlugin` object with `name: 'sprite-commit'` and `configureServer(server)` hook
   - Register middleware at path `/api/sprite-commit` accepting POST requests
   - Request body (JSON): `{ name: string, pngBase64: string, manifest: SpriteManifestEntry }`
   - Handler actions:
     a. Create directory `public/sprites/${name}/` if not exists (using `fs.mkdirSync` with `{ recursive: true }`)
     b. Write PNG: decode base64, write to `public/sprites/${name}/sheet.png` (using `fs.writeFileSync`)
     c. Read existing sprites.json: `JSON.parse(fs.readFileSync('public/sprites/sprites.json', 'utf-8'))`
     d. Add/update entry: `manifest[name] = requestBody.manifest`
     e. Write updated sprites.json: `fs.writeFileSync('public/sprites/sprites.json', JSON.stringify(manifest, null, 2))`
     f. Return `{ ok: true, path: \`sprites/${name}/sheet.png\` }`
   - Add plugin to the `plugins` array in defineConfig
   - Import `fs` from `node:fs` and `path` from `node:path` at top of vite.config.ts

2. Create src/viewer/commit.ts:
   - Export async function `commitSprite(name: string, file: File, frameWidth: number, frameHeight: number, definedRanges: Array<{name: string, start: number, end: number, fps: number}>)`:
     - Convert File to base64: read as ArrayBuffer, convert to base64 string
     - Build SpriteManifestEntry: `{ sheet: \`${name}/sheet.png\`, frameWidth, frameHeight, animations: {} }`
     - Populate animations from definedRanges: for each range, `animations[range.name] = { frames: [range.start, range.end], frameRate: range.fps }`
     - If no ranges defined, create a default 'idle' animation with frames [0, 0]
     - POST to `/api/sprite-commit` with JSON body `{ name, pngBase64, manifest: entry }`
     - Return response JSON
   - Export function `fileToBase64(file: File): Promise<string>` helper
